# Temantiket Mobile Design System & Page Redesign Plan

> Goal: transform Temantiket mobile into a modern travel super-app experience inspired by Trip.com style: clean, blue-white, fast, trustworthy, card-based, and easy for non-technical users.

---

## 1. Design Direction

### Brand feeling
Temantiket should feel like:

- modern travel assistant
- trusted student/traveler service
- fast booking and document management app
- premium but still friendly
- Indonesian + Egypt travel specialist

### Visual keywords

- Blue travel super-app
- Rounded cards
- Big hero area
- Clear search/action bar
- Icon service grid
- Soft shadows
- Compact mobile typography
- Bottom navigation
- Minimal clutter
- Data shown as cards, not raw admin tables

### Mobile reference style

The target mobile home/dashboard should look closer to a consumer travel app than an admin panel:

- greeting header
- search bar
- service shortcut grid
- promo/action cards
- recent activity cards
- stats as small friendly cards
- floating/bottom navigation

## 4. Global Mobile Rules

### Layout rules

- No horizontal page overflow.
- Every wide list/table must be inside `overflow-x-auto`.
- Cards must use `min-w-0` inside flex rows.
- Long emails/URLs must wrap with `overflow-wrap:anywhere`.
- Avoid raw tables on mobile; use cards where possible.

### Interaction rules

- Tappable cards should have `active:scale-[0.98]`.
- Buttons minimum height: 40px.
- Icon shortcut minimum tap area: 44px.
- Bottom nav must not cover inputs when keyboard opens.

### Responsive rule

- Desktop layout must remain unchanged unless explicitly redesigning desktop.
- Mobile redesign should be guarded with `md:hidden` for new mobile sections and `hidden md:block` for old desktop where needed.

## Phase 18 — AITEM Chat Widget Mobile

### Target
Chat should feel polished and never break layout.

### Fix requirements

- AI markdown tables must be horizontally scrollable.
- Emails and URLs must wrap or stay inside table scroll.
- Bubble max width safe.
- Assistant content should use responsive markdown renderer.
- Mobile chat modal should have controlled width and no page overflow.

### Table styling

```tsx
table: ({ children }) => (
  <div className="my-2 max-w-full overflow-x-auto rounded-xl border border-slate-200 bg-white">
    <table className="w-max min-w-full border-collapse text-[11px] md:text-xs">
      {children}
    </table>
  </div>
)
```

`td` should include:

```txt
max-w-[160px] md:max-w-[220px] break-words [overflow-wrap:anywhere]
```

# 9. Rules for Replit AI / Implementation Agent

When implementing this design:

1. Do not change auth logic unless explicitly asked.
2. Do not change Supabase schema unless explicitly asked.
3. Do not delete data.
4. Do not recreate agency/member/client/order.
5. Do not apply huge global CSS overrides that affect desktop.
6. Prefer reusable components.
7. Keep desktop unchanged at first.
8. Every changed page must pass lint/test/build.
9. Every new mobile component must handle long text.
10. Every wide table must be inside horizontal scroll.
11. If a page is complex, redesign mobile only first.
12. Report all changed files and risks.

---

# 10. Acceptance Criteria

The mobile redesign is considered successful when:

- first screen after login looks like a travel app, not raw admin dashboard
- user can access main actions within 1 tap from dashboard
- dashboard shows real old data correctly
- agent/staff list appears correctly
- no mobile horizontal overflow
- AI chat tables no longer break layout
- public ticket and booking pages feel trustworthy
- desktop still works
- production audit stays clean

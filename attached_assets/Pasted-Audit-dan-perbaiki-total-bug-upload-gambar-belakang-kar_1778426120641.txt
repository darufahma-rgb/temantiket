Audit dan perbaiki total bug upload gambar belakang kartu staff/member/agent yang terus gagal simpan ke database dengan error “The page could not be found NOT_FOUND fra1/sin1::...”. Jangan hanya tambal UI atau ganti toast; cari akar masalah sampai tuntas. Telusuri seluruh flow upload dari tombol “Upload Gambar Belakang Kartu”, file input, upload ke Supabase Storage, generate public/signed URL, update database, fetch ulang data, render kartu, sampai refresh halaman. Error `fra1::...` atau `sin1::...` terlihat seperti request/page id dari platform, bukan ID database, jadi pastikan sistem tidak memakai route slug, temporary page id, request id, atau object preview id untuk update database. Gunakan primary key database yang benar seperti `profile.id`, `staff.id`, `agent.id`, `member.id`, atau `user_id` sesuai tabel yang benar-benar dipakai kartu tersebut.

Pastikan database punya kolom permanen untuk menyimpan URL gambar belakang, misalnya `card_back_image_url`, dan migration SQL sudah benar. Cek table mana yang sebenarnya dipakai oleh staff card, agent card, dan member card; jangan update tabel yang salah. Jika record belum ada, gunakan upsert record yang valid. Audit RLS policy agar owner/admin bisa update semua kartu, sedangkan user biasa hanya bisa update kartunya sendiri jika memang diizinkan. Setelah upload storage berhasil, update database harus berhasil dulu baru tampilkan success. Kalau database update gagal, hapus file storage yang baru diupload agar tidak orphan file.

Tambahkan `/api/health-check` atau preflight check sebelum upload untuk memastikan Supabase URL, service role/admin client, database, storage bucket, table, column, dan permission UPDATE semuanya valid. Tambahkan logging detail server/client untuk `tableName`, `recordId`, `userId`, `role`, `bucket`, `storagePath`, `publicUrl`, `updatePayload`, dan error asli Supabase/API. Jangan tampilkan error generik saja. Setelah diperbaiki, gambar belakang kartu harus tersimpan permanen, tetap muncul setelah refresh, login ulang, beda device, dan tidak ada lagi error NOT_FOUND. Buat laporan singkat dalam bahasa Indonesia: akar masalahnya apa, file/flow apa yang diperbaiki, dan bukti test berhasil.
